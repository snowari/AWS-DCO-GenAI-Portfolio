# 교육용 DCO SOP 요약 및 체크리스트

*본 문서는 실제 AWS 내부 SOP가 아니라 교육용 샘플 SOP를 바탕으로 작성한 학습 자료입니다. 실제 장비 점검, 실제 네트워크 진단, 실제 AWS 환경 접속을 요구하지 않습니다.*

---

## 1. SOP를 목적·절차·주의사항·확인항목으로 나누기

### 1. 이 SOP의 목적

이 SOP는 교육용 샘플 로그에서 `Link Down` 이벤트가 보였을 때, 학생이 발생 시간, 샘플 장비명, 반복 여부, Ticket 관련 표현, Escalation 필요 여부를 정리하는 연습을 하기 위한 문서입니다. 실제 장비를 점검하는 절차가 아니라, 샘플 로그를 읽고 관찰 내용을 안전하게 기록하는 학습용 SOP입니다.

### 2. 절차 요약

1. 샘플 로그에서 `Link Down` 이벤트가 표시된 시간을 확인합니다.
2. 이벤트에 표시된 `SAMPLE`, `DEMO`, `EDU` 형식의 샘플 장비명을 확인합니다.
3. 같은 샘플 장비에서 반복 이벤트가 있는지 확인합니다.
4. `Ticket opened` 또는 `Ticket escalated` 같은 표현이 있는지 확인합니다.
5. 영향 범위를 단정하지 않고, 로그에 나타난 관찰 내용만 정리합니다.
6. 혼자 판단하기 어려운 내용은 Escalation 필요 여부로 표시합니다.
7. 확인한 내용을 Handover Memo 또는 Incident Report 초안에 넣을 수 있도록 짧게 정리합니다.

### 3. 주의사항

- 실제 장비에 접속하지 않습니다.
- 실제 네트워크 진단 명령을 실행하지 않습니다.
- 실제 IP 주소, 실제 티켓 번호, 실제 장비 시리얼을 사용하지 않습니다.
- AI 답변이 원인을 하나로 단정하면 학생이 직접 수정해야 합니다.
- 교육용 샘플 로그에 나온 내용만 근거로 작성해야 합니다.

### 4. 마지막 확인항목

□ 발생 시간이 있는가?  
□ 샘플 장비명이 있는가?  
□ 반복 여부를 확인했는가?  
□ 관련 Ticket 표현이 있는가?  
□ Escalation 필요 여부를 표시했는가?  
□ 실제 정보가 포함되지 않았는가?  
□ AI 답변에서 원인 단정이나 실제 절차처럼 보이는 표현을 수정했는가?
