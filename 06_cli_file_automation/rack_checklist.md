# 교육용 샘플 데이터 사용

# Rack Log Review Checklist

## 1. 발생 시간 확인

□ 샘플 로그에서 CRC error 증가가 표시된 시간을 확인합니다.  
□ 샘플 로그에서 Link Down 이벤트가 표시된 시간을 확인합니다.  
□ 두 이벤트의 발생 순서를 교육용 메모로 정리합니다.

## 2. 샘플 장비명 확인

□ 이벤트에 표시된 샘플 장비명이 `SAMPLE` 또는 `EDU` 접두어를 사용하는지 확인합니다.  
□ 샘플 장비명이 `SAMPLE_TOR_SW_01`로 일관되게 기록되어 있는지 확인합니다.  
□ 실제 장비명처럼 보이는 정보가 포함되어 있지 않은지 확인합니다.

## 3. 반복 여부 확인

□ CRC error 증가 표현이 한 번만 보이는지 반복적으로 보이는지 확인합니다.  
□ Link Down 이벤트가 같은 샘플 장비명에서 반복되는지 확인합니다.  
□ 반복 여부는 원인 단정이 아니라 관찰 내용으로만 기록합니다.

## 4. Ticket 상태 확인

□ 샘플 티켓에 티켓 ID가 교육용 형식으로 기록되어 있는지 확인합니다.  
□ Ticket 상태 또는 Escalation 필요 여부가 표시되어 있는지 확인합니다.  
□ 실제 티켓 번호나 내부 시스템 번호가 포함되어 있지 않은지 확인합니다.

## 5. 원인 단정 금지

□ CRC error와 Link Down의 원인을 하나로 확정하지 않습니다.  
□ 케이블, 포트, 장비 문제 등 실제 원인을 추측해 단정하지 않습니다.  
□ 샘플 로그에 보이는 사실만 기록합니다.
