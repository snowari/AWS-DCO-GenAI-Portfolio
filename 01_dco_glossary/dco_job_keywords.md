# [교육용 샘플] DCO 직무 키워드 정의서

*본 문서는 폴리텍대학 AX과정 교육 실습을 위해 가상의 데이터센터 운영 정보를 기초로 작성한 포트폴리오 산출물입니다.*

## 1. DCO 핵심 직무 키워드 5대 요약

1. **하드웨어 트러블슈팅 (Hardware Troubleshooting)**
   - 쉬운 뜻: 서버나 네트워크 장비에서 RAM, SSD, 전원 공급 장치, 쿨러 팬 같은 부품에 문제가 생겼을 때 원인을 확인하고 조치하는 업무입니다. 고장 부품을 임의로 만지는 것이 아니라 정해진 절차와 티켓 기록에 따라 점검합니다.
   - 준비 각오: 서버 기본 부품의 역할과 장애 발생 시 확인해야 할 기본 절차를 꼼꼼히 공부하겠습니다.

2. **광케이블 커넥터 세척 (Fiber Connector Cleaning)**
   - 쉬운 뜻: 데이터 신호가 지나가는 광케이블 연결 단자에 먼지나 이물질이 묻으면 통신 품질이 떨어질 수 있습니다. 전용 도구와 절차를 사용해 커넥터를 깨끗하게 관리하는 작업입니다.
   - 준비 각오: 광 패치 코드 취급 방법과 세척 SOP 규정을 철저히 지키겠습니다.

3. **자산 관리 (Asset Management)**
   - 쉬운 뜻: 서버, 부품, 케이블, 랙 위치 같은 물리 자산의 위치와 상태를 정확히 기록하고 추적하는 업무입니다. 어떤 장비가 어디에 있고 언제 설치, 교체, 회수되었는지 관리하는 것이 중요합니다.
   - 준비 각오: 자산명, 수량, 위치, 상태를 실수 없이 입력하는 습관을 기르고 엑셀 기본 기능을 익히겠습니다.

4. **물리 보안 규칙 (Physical Security Rules)**
   - 쉬운 뜻: 데이터센터는 중요한 장비와 고객 데이터가 있는 공간이므로 출입, 촬영, 장비 접촉, 동행 규칙이 엄격합니다. 허가받지 않은 행동이나 정보 공유는 보안 사고로 이어질 수 있습니다.
   - 준비 각오: 출입증 관리, 사진 촬영 금지, 보안 구역 동행 원칙을 숙지하고 현장 지시를 정확히 따르겠습니다.

5. **장애 대응 티켓 (Incident Ticket Handling)**
   - 쉬운 뜻: 장비 이상이나 작업 요청이 생기면 티켓이라는 기록 단위로 접수하고 처리 과정을 남깁니다. 문제 상황, 조치 내용, 결과를 정확히 기록해야 다른 담당자도 상황을 이해할 수 있습니다.
   - 준비 각오: 문제 상황, 확인 내용, 조치 결과, 다음 단계를 짧고 명확하게 정리하는 연습을 하겠습니다.

## 2. 1차시 실습 소감
- 생성형 AI 비서와 직무에 대해 대화해보니, DCO 직무가 소프트웨어 코딩에만 국한되지 않고 하드웨어를 꼼꼼히 관리하며 물리적 신뢰성과 보안 절차를 지키는 일이 중요함을 알게 되었습니다.
